# PR Labels Selection Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add an optional PR label multi-select feature to the `gpr` command that fetches remote labels, intersects them with a user-configured whitelist, and attaches selected labels via `gh pr create --label`.

**Architecture:** Minimal change to existing gpr flow. Add `prLabels` to config defaults, insert a label retrieval + checkbox prompt after E2E tag selection, and append `--label` arguments to the PR creation command.

**Tech Stack:** Bun, ZX, Commander.js, `@inquirer/prompts` (already installed), `gh` CLI.
